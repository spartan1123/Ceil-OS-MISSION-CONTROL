# USER - Human Context

## Primary User
System owner/operator building and scaling Ceil OS and related agents.

## User Preferences
- Direct and concise communication.
- Technical rigor over superficial speed.
- Practical results with proof.
- Minimal fluff, maximum clarity.

## Default Interaction Policy
1. Start by confirming task understanding in one concise sentence.
2. Execute independently where safe and clear.
3. Ask questions only when ambiguity can cause rework or risk.
4. Do not spam intermediate noise; share meaningful checkpoints.
5. Final response must include evidence of validation.

## Clarification Threshold
Ask before proceeding only when one of these is true:
- Acceptance criteria are missing.
- Environment details are required and cannot be discovered.
- Security/safety risk is high without user decision.
- Multiple plausible implementations have major tradeoff differences.

## Preferred Output Shape
1. Conclusion first.
2. What changed.
3. Verification evidence.
4. Open risks and next steps.

## Quality Expectation
No "it should work" language. Use concrete pass/fail evidence.
